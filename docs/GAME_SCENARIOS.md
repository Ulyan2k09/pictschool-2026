# Игровые сценарии MVP

Этот документ описывает базовый игровой слой: карту, цель, задания, вмешательства ИИ и ожидаемое поведение системы. Его задача — дать кураторам единый сценарий, который можно реализовывать, тестировать и усложнять.

## Стартовая карта MVP

Для MVP используется поле `8 x 6`.

- Старт платформы: `{ "x": 0, "y": 0 }`.
- Цель миссии: `{ "x": 7, "y": 5 }`.
- Постоянное препятствие: `wall-1` в `{ "x": 3, "y": 2 }`.
- Перемещаемый объект: `box-1` в `{ "x": 2, "y": 2 }`.
- Дополнительный объект задания: `task-item-1` в `{ "x": 5, "y": 3 }`.

Карта должна быть достаточно простой, чтобы команда могла понять ее за несколько минут, но на ней должен существовать короткий маршрут, который ИИ может заблокировать.

## Цель игрока

Игрок должен провести платформу от старта до цели, выполнить минимум одно задание на поле и сохранить систему в рабочем состоянии. Победа засчитывается, если платформа дошла до цели, миссия не провалена, а обязательное задание выполнено.

## Базовые задания

- Дойти до контрольной точки.
- Подтвердить выполнение действия через мобильное приложение.
- Обойти или обработать препятствие.
- Завершить маршрут в зоне цели.

## Сценарий 1. Нормальный проход без вмешательства ИИ

1. Миссия запускается через `POST /api/mission/start`.
2. Mobile App получает поле, стартовую позицию, объекты и цель.
3. Игрок отправляет команды движения.
4. Платформа обновляет позицию через события `platform.updated`.
5. При достижении задания бэкенд меняет очки через `mission.scored`.
6. При достижении цели бэкенд публикует `mission.completed`.

Ожидаемый результат: миссия завершается успешно, журнал событий объясняет маршрут и начисление очков.

## Сценарий 2. ИИ передвигает объект и блокирует короткий маршрут

1. Игрок выбирает короткий маршрут к цели.
2. ИИ получает состояние и выбирает действие `move_object`.
3. Бэкенд проверяет, можно ли переместить `box-1` в новую позицию.
4. Объект перемещается в клетку короткого маршрута.
5. Бэкенд публикует `ai.object_moved` и, если путь закрыт, `ai.path_blocked`.
6. Mobile App показывает, какой объект перемещен и почему движение стало сложнее.

Ожидаемый результат: игрок может построить альтернативный маршрут, а система не становится гарантированно непроходимой.

## Сценарий 3. Невозможная команда движения

1. Игрок отправляет `move_forward`.
2. Следующая клетка занята объектом или препятствием.
3. Бэкенд отклоняет команду с ошибкой `path_blocked`.
4. Платформа не должна начинать опасное движение.
5. Mobile App показывает понятное сообщение об ошибке.

Ожидаемый результат: состояние миссии остается консистентным, журнал фиксирует причину отказа.

## Сценарий 4. Ошибка платформы или симуляции

1. Бэкенд отправляет команду движения.
2. Платформа возвращает статус `error` или `blocked`.
3. Бэкенд публикует `platform.updated`.
4. Mobile App показывает ошибку и доступное действие восстановления.
5. Команда может продолжить после восстановления или миссия завершается `mission.failed`.

Ожидаемый результат: ошибка не теряется и не выглядит как зависание приложения.

## Сценарий 5. Reset перед новым прогоном

1. Куратор вызывает `POST /api/mission/reset`.
2. Бэкенд переводит миссию в `idle`.
3. Объекты возвращаются в стартовые позиции.
4. Платформа возвращается в безопасное стартовое состояние или симуляция сбрасывает координаты.
5. Новый запуск снова начинается с одинаковых условий.

Ожидаемый результат: разные команды проходят сопоставимый сценарий.

## Победа, проигрыш и баланс

Победа:

- платформа достигла цели;
- обязательное задание выполнено;
- миссия не находится в статусе `failed`.

Проигрыш:

- время вышло;
- платформа перешла в невосстановимую ошибку;
- команда нарушила критическое правило сценария;
- ИИ создал препятствие, а команда не смогла восстановить маршрут в рамках времени.

Баланс:

- ИИ не должен ставить объект так, чтобы не осталось ни одного возможного маршрута, если это не отдельный финальный режим.
- Каждое жесткое вмешательство ИИ должно быть видно в интерфейсе и журнале.
- У команды должен быть хотя бы один способ восстановить управление или перестроить маршрут.
