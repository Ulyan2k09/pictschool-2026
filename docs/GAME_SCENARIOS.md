# Игровые сценарии MVP

Документ описывает базовые сценарии для новой механики:

- на поле одновременно находятся `agent` и `robot`;
- цель — собрать как можно больше уточек;
- участники ходят по очереди;
- у каждого в ходе до `5` движений;
- раунд завершается, когда все уточки собраны.

## Стартовая карта MVP

Для MVP используется поле `10 x 10`.

- Старт `robot`: `{ "x": 0, "y": 0 }`, направление `E`.
- Старт `agent`: `{ "x": 9, "y": 9 }`, направление `W`.
- Уточки: 10 штук в заранее фиксированных клетках.
- Препятствия: минимум 4 постоянных блока.

## Сценарий 1. Базовый раунд без ошибок

1. Раунд запускается через `POST /api/round/start`.
2. Активный участник (`robot`) отправляет до 5 команд.
3. Backend отправляет в симуляцию TCP-строку, например: `1 1 3 1`.
4. При успешной TCP-отправке backend применяет ход локально по правилам поля.
5. Backend пишет `turn.completed` и переключает ход на `agent`.
6. Раунд продолжается, пока все уточки не собраны.

Ожидаемый результат: раунд корректно доходит до `round.completed`.

## Сценарий 2. Сбор уточки за ход

1. Участник получает ход и отправляет пакет команд.
2. По итоговой траектории симуляция фиксирует пересечение клетки с уточкой.
3. Backend увеличивает счет этого участника.
4. Публикуется событие `duck.collected`.

Ожидаемый результат: счет и число оставшихся уточек согласованы между состоянием и журналом.

## Сценарий 3. Превышение лимита движений

1. Активный участник отправляет 6 команд.
2. Backend отклоняет запрос без отправки в симуляцию.
3. Возвращается ошибка `turn_limit_exceeded`.

Ожидаемый результат: состояние поля и очередность хода не меняются.

## Сценарий 4. Неизвестная команда

1. Активный участник отправляет `commands: [1, 7, 4]`.
2. Backend валидирует и отклоняет пакет.
3. Возвращается ошибка `unknown_command`.

Ожидаемый результат: в симуляцию не отправляется ни один TCP-пакет.

## Сценарий 5. Ошибка симуляции

1. Backend отправляет корректную TCP-строку на командный порт.
2. Симуляция отвечает ошибкой (например, недопустимое состояние).
3. Backend фиксирует `turn.failed` и отдает `simulation_error`.

Ожидаемый результат: ошибка видна в API и журнале, раунд остается консистентным.

## Сценарий 6. Завершение раунда

1. Перед ходом на поле осталась одна уточка.
2. Активный участник собирает ее в текущем ходе.
3. Backend обновляет `ducksLeft` до `0`.
4. Backend публикует `round.completed`.

Ожидаемый результат: победитель определяется по количеству собранных уточек.
