# Критерии готовности MVP

Этот чеклист нужен кураторам перед стартом школы и перед финальным соревнованием. Если пункт нельзя проверить вручную за несколько минут, его нужно упростить или добавить отдельный тестовый сценарий.

## Перед стартом школы

- MVP запускается по инструкции без ручной настройки кода.
- Есть стартовая карта, цель, минимум один объект и минимум одно задание.
- `GET /api/mission` возвращает состояние миссии.
- `POST /api/mission/start` запускает прогон.
- `POST /api/player/command` принимает команду игрока.
- `POST /api/ai/action` принимает действие ИИ.
- `GET /api/events` возвращает журнал событий.
- `POST /api/mission/reset` возвращает систему к стартовым условиям.
- Mobile App показывает поле, платформу, объекты и состояние миссии.
- Platform / Simulation возвращает позицию, статус и ошибки.

## Готовность backend

- Бэкенд является единственным источником правды.
- Все команды игрока валидируются до отправки платформе.
- Все действия ИИ валидируются до изменения поля.
- Ошибки возвращаются в едином формате.
- События объясняют ход миссии.
- Reset воспроизводимо возвращает стартовые позиции.

## Готовность Mobile

- Игрок видит поле, цель, платформу и объекты.
- Игрок видит, какой объект передвинул ИИ.
- Игрок видит заблокированный путь или недоступное действие.
- Ошибки не выглядят как зависание.
- Команды управления доступны в финальном сценарии без лишних экранов.

## Готовность AI

- ИИ получает состояние миссии от бэкенда.
- ИИ умеет выполнить минимум одно действие `move_object`.
- ИИ не меняет состояние напрямую.
- У действия ИИ есть `reason`.
- Вмешательство усложняет маршрут, но не делает миссию нечестно невозможной.

## Готовность Platform / Simulation

- Платформа или симуляция принимает базовые команды движения.
- Возвращает `ready`, `moving`, `blocked`, `error` или `recovering`.
- Не выполняет движение в занятую или заблокированную клетку.
- Поддерживает безопасный reset.
- Ошибки воспроизводимы и видны в журнале.

## Перед финалом

- Финальный сценарий можно прогнать несколько раз в одинаковых стартовых условиях.
- Команды проходят сопоставимые условия.
- ИИ действительно вмешивается через объекты и маршрут.
- Event log позволяет объяснить, почему команда победила или проиграла.
- Куратор может быстро перезапустить миссию после сбоя.
- Есть понятные критерии победы, проигрыша и начисления очков.
