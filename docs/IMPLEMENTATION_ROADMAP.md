# Roadmap реализации MVP

Порядок внедрения под новый контракт: команды в симуляцию отправляются строкой байт по TCP, игра идет раундами с очередностью ходов.

## Этап 1. Round state и event log

Сначала собрать backend как источник правды.

Готово, когда:

- есть состояния `idle`, `running`, `completed`, `failed`;
- есть модель поля, уточек и двух участников (`robot`, `agent`);
- `POST /api/round/start` создает стартовое состояние;
- `GET /api/round` возвращает состояние;
- каждое изменение пишется в журнал;
- `GET /api/events` возвращает журнал.

## Этап 2. TCP-интеграция с симуляцией

Подключить транспорт команд через localhost TCP.

Готово, когда:

- backend формирует строку вида `1 2 3 4`;
- отправка идет на `SIM_TCP_COMMAND_PORT=5055`;
- в конфиге зафиксирован резерв `SIM_TCP_TELEMETRY_PORT=5056`;
- ошибки симуляции превращаются в `simulation_error`;
- есть повторяемый reset после ошибок.

## Этап 3. Turn API и валидация

Реализовать API хода и правила ограничений.

Готово, когда:

- `POST /api/turn/submit` принимает `actor` и `commands`;
- валидируется очередь хода (`activeActor`);
- валидируется длина `commands` (`1..5`);
- валидируются только коды `1..4`;
- успешный ход публикует `turn.completed`.

## Этап 4. Подсчет уточек и завершение раунда

Добавить игровую цель и завершение.

Готово, когда:

- сбор уточек обновляет счет участника;
- публикуется `duck.collected`;
- `ducksLeft` обновляется после каждого хода;
- при `ducksLeft == 0` публикуется `round.completed`.

## Этап 5. Клиент и live-обновления

Подключить отображение в клиенте.

Готово, когда:

- клиент видит активного участника, счет и уточки;
- `GET /api/live` отдает события (SSE или WebSocket);
- клиент обновляет экран без ручного refresh;
- после переподключения состояние восстанавливается через `GET /api/round`.

## Этап 6. Интеграционные проверки

Собрать весь поток в повторяемый прогон.

Готово, когда проходят сценарии:

- корректный ход с 1-5 командами;
- ошибка при 6 командах (`turn_limit_exceeded`);
- ошибка при неизвестной команде (`unknown_command`);
- корректная обработка `simulation_error`;
- завершение раунда при сборе последней уточки.

## Минимальный порядок для кураторов

1. Зафиксировать карту и позиции уточек.
2. Реализовать round state и event log.
3. Подключить TCP-отправку команд в симуляцию.
4. Добавить turn API и валидацию.
5. Добавить подсчет очков и завершение раунда.
6. Подключить live-обновления в клиенте.
7. Провести интеграционный прогон.
