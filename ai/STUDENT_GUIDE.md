# Руководство школьника: Трек ИИ

Привет! Добро пожаловать в трек ИИ.

В этой задаче наша цель — запрограммировать "мозг" ИИ-агента, который будет играть против робота, собирать уточек на поле и обходить препятствия.

Наш агент не просто пишет случайные числа, он использует нейросеть (LLM, большую языковую модель), чтобы оценивать обстановку и принимать осознанные решения.

## Как устроена система

Весь проект состоит из нескольких частей, которые общаются друг с другом:

1.  **Бэкенд (Судья)** — хранит карту, счет, кто сейчас ходит, и проверяет, не нарушены ли правила.
2.  **Эмулятор симуляции (Исполнитель)** — "двигает" модельки по полю.
3.  **Визуализатор (Экран)** — показывает нам поле и игру в браузере.
4.  **Агент ИИ (Мозг)** — наш Python-скрипт. Он смотрит на поле и решает, куда идти.

## Шаг 1: Подготовка к запуску

Сначала нужно настроить Python и доступы к нейросети.

1.  **Открой терминал** и перейди в папку с агентом:
    ```bash
    cd ai
    ```

2.  **Создай виртуальное окружение** и установи библиотеки:
    ```bash
    python3 -m venv .venv
    source .venv/bin/activate
    pip install -r requirements.txt
    ```

3.  **Настрой ключи доступа (env-файл)**:
    Скопируй шаблон конфигурации в рабочий файл:
    ```bash
    cp .env.example .env
    ```
    Открой файл `.env` в редакторе и вставь выданный куратором токен в строку:
    `OPENAI_API_KEY=твой_ключ_здесь`

## Шаг 2: Запуск игры

Для игры нам нужно поднять все компоненты системы. Вернись в главную папку проекта (где лежит папка `ai`):

1.  **Запусти серверы (Бэкенд + Эмулятор + Агент)**
    В терминале выполни одну волшебную команду:
    ```bash
    ./infrastructure/manual/start-backend-ai-stack.sh
    ```
    *Терминал будет занят, оставь его открытым! Если хочешь остановить серверы, нажми `Ctrl+C`.*

2.  **Запусти визуализатор (UI)**
    Открой **новую вкладку** терминала в главной папке проекта и запусти:
    ```bash
    ./infrastructure/manual/start-ai-visualizer.sh
    ```

3.  **Открой игру в браузере**
    *   **Визуализатор (Сама игра):** перейди по ссылке [http://127.0.0.1:5174](http://127.0.0.1:5174).
    *   **API Документация (Шпаргалка по командам):** перейди по ссылке [http://127.0.0.1:8080/api/docs](http://127.0.0.1:8080/api/docs).

### Как играть:
В браузере (на порту 5174) нажми "Connect", затем "New round".
Ты играешь за **Робота (Robot)**. Нажми кнопки `Forward`, `Left` и т.д. (до 5 команд), затем нажми "Submit robot turn".
Как только ты сходишь, **Агент (Agent)** автоматически отправит свой ход бэкенду, и ты увидишь результат на экране!

## Шаг 3: Как работает мозг агента

Открой файл `ai/agent_service/planner.py`. Это самое главное место!

Что делает этот код:
1.  Читает текущую карту (где Робот, где Агент, где Утки и Стены).
2.  Собирает из этого **Промпт** (`_build_prompt`) — текстовое описание для нейросети.
3.  Отправляет промпт в LLM.
4.  LLM возвращает список команд (от 1 до 5 команд. `1`-вперед, `2`-назад, `3`-влево, `4`-вправо) и своё объяснение (`rationale`), почему она так решила.
5.  Если интернет отпал или нейросеть сломалась, срабатывает `fallback_commands_for_round` — простой алгоритм, который всегда поворачивается или идет к ближайшей утке.

## 🚀 Практические задания (Попробуй сам!)

Выбери уровень сложности и начни менять код в `ai/agent_service/planner.py`. После каждого изменения останавливай агент (`Ctrl+C` в терминале) и запускай его заново!

### Уровень 1: Разведчик (Легко)
*   **Задача:** Изменить характер агента через промпт.
*   **Действие:** Найди в `planner.py` системный промпт (там где написано `"role": "system"` и `"You are a tactical game agent..."`). Добавь туда новое правило!
    Например: *"Ты трусливый агент. Старайся держаться подальше от робота"*. Или *"Ты жадный агент, собирай уток любой ценой, даже если врежешься в стену"*.
*   **Проверка:** Запусти игру и посмотри, как ведет себя агент. В терминале, где работает скрипт агента, печатается `rationale` — почитай, как агент объясняет свои действия!

### Уровень 2: Аналитик (Средне)
*   **Задача:** Сейчас агент получает в промпт список координат **вообще всех** уток и препятствий. Когда объектов на поле станет очень много, нейросеть запутается в этих координатах и потратит много токенов.
*   **Действие:** Измени функцию `_build_prompt` в `planner.py`. Сделай так, чтобы агент получал только координаты **ближайших 3-х уток**, а не всех подряд.
    *(Подсказка: чтобы найти ближайшую утку, используй математику. Формула манхэттенского расстояния: `abs(x1 - x2) + abs(y1 - y2)`).*
*   **Проверка:** Агент стал выдавать команды быстрее? Он по-прежнему идет к уткам, а не в пустой угол?

### Уровень 3: Инженер (Сложно)
*   **Задача:** Научить агента не врезаться в стены при сложных маневрах. Сейчас LLM может выдать 5 команд подряд `[1, 1, 1, 1, 1]`, но на пути может быть стена уже на втором шаге.
*   **Действие:** Напиши свой собственный мини-симулятор шагов внутри Python-кода планера.
    Сначала спроси у LLM план (список команд). Затем, с помощью циклов `for` и координат стен, проверь: не врежется ли агент в стену, если выполнит этот план?
    Если план плохой (врезается) — отбрасывай ответ LLM и используй безопасный ход из `fallback_commands_for_round`.
*   **Проверка:** Построй в голове ловушку (например, П-образную стену). Справится ли твой код защиты от столкновений?
